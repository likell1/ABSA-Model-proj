# ABSA 라벨링 검수 가이드라인 (간소화 버전)

## 1. 핵심 원칙

**"텍스트에 명시적으로 있는 것만 인정한다"**

---

## 2. 필수 검수 항목

### 2.1 Term 검수

| 규칙 | 예시 |
|------|------|
| 원문에 **있는 그대로** 추출 | "편해요" → term: "편해요" (O) / "착용감" (X) |
| 추론된 term 삭제 | 위 예시에서 "착용감"은 원문에 없음 → 삭제 |

### 2.2 Evidence 검수

| 규칙 | 예시 |
|------|------|
| evidence는 **감성 표현 포함** 필수 | "디자인이 예뻐요" → evidence: "디자인이 예뻐요" (O) / "디자인" (X) |
| evidence_span이 null이면 **삭제 대상** | 실제 위치 특정 불가 = 추론된 것 |

### 2.3 Category 검수

| 상황 | 조치 |
|------|------|
| term과 category 불일치 | category 수정 |
| 애매하면 | 가장 직관적인 것 선택 |

---

## 3. 검수 예시

**원문:** "편하고 디자인이 예뻐요 가격도 좋아요 시원해요 빨리 마르고 이것만 입게되요"

### 3.1 LLM 출력 검수

| 원본 term | 조치 | 이유 |
|-----------|------|------|
| 디자인 | ✅ 유지 (evidence 수정) | evidence → "디자인이 예뻐요" |
| 가격 | ✅ 유지 (evidence 수정) | evidence → "가격도 좋아요" |
| 착용감 | ❌ **삭제** | 원문에 "착용감" 없음, evidence_span도 null |

### 3.2 추가해야 할 항목

| term | category | evidence |
|------|----------|----------|
| 편하고 | 착용감 | 편하고 |
| 시원해요 | 기능성 | 시원해요 |
| 빨리 마르고 | 기능성 | 빨리 마르고 |

---

## 4. 빠른 검수 체크리스트

```
□ evidence_span이 null인가? → 삭제 검토
□ term이 원문에 그대로 있는가? → 없으면 삭제/수정
□ evidence에 감성표현이 포함되어 있는가? → 없으면 확장
```

---

## 5. Category 목록

| Category | 포함 내용 |
|----------|----------|
| 착용감 | 편함, 불편함, 핏 |
| 디자인/스타일 | 예쁨, 색상, 모양 |
| 가격 | 비쌈, 저렴함, 가성비 |
| 품질 | 소재, 내구성, 마감 |
| 기능성 | 시원함, 따뜻함, 속건성 등 |
| 배송 | 빠름, 느림, 포장 |
| 사이즈 | 큼, 작음, 정사이즈 |

---

## 6. Polarity 기준

| 값 | 의미 | 예시 |
|----|------|------|
| 1 | 긍정 | 예뻐요, 편해요, 좋아요 |
| 0 | 중립 | 보통이에요, 평범해요 |
| -1 | 부정 | 별로예요, 불편해요, 비싸요 |

---

## 7. 주의사항

1. **과도한 추론 금지**: LLM이 문맥상 추론한 aspect는 삭제
2. **중복 방지**: 같은 표현에 여러 aspect 부여 금지
3. **일관성 유지**: 팀원 간 동일 기준 적용
4. **애매한 경우**: 슬랙/노션에 질문 후 결정
